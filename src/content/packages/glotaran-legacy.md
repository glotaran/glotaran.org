If you already use the original Glotaran desktop application, start here.
This page collects the most useful entry points for the Java GUI so existing
users do not have to dig through the archived website structure.

## Existing users start here

- Open the [legacy resources page](/packages/glotaran-legacy/resources) for a
  curated entry point to the archived documentation and original screencast.
- Use the full [legacy documentation site](https://glotaran.github.io/legacy/)
  when you need installation notes, workflow details, file formats, or other
  archived reference material.
- Download binaries from the
  [GitHub releases page](https://github.com/glotaran/glotaran-legacy/releases).
- Cite the original Java UI with the [2012 Glotaran paper](/publications).

Glotaran 1.x, maintained in this repository as `glotaran-legacy`, was the
first graphical user interface for TIMP. It was built on the NetBeans Platform
and runs on any operating system with Java 8.

The software was first released in 2011 and described in the
[2012 JSS paper](/publications). It is no longer under active development,
but the codebase is maintained on GitHub and the 1.5.1 release remains
functional for users who prefer a standalone desktop application.

**New users should start with [pyglotaran](/packages/pyglotaran)**, which provides
the same analytical capabilities with a modern Python interface, active
development, and better integration with the scientific software ecosystem.

## What is preserved

The redesigned site now keeps legacy Java UI support material close to this

## Download

Version 1.5.1 is available from the
[GitHub releases page](https://github.com/glotaran/glotaran-legacy/releases).

Requires Java 8 or later. No installation needed — run the `.jar` directly.

## Still useful today

The legacy application remains relevant for users who already have established
analysis workflows around the desktop GUI. The most useful preserved material
from the old site is:

- the archived documentation for the Java UI,
- the original screencast showing a full analysis workflow,
- and the publication record describing the scientific background.

That material is kept available for continuity, while new feature development
and new-user guidance now live in the Python-based tooling around pyglotaran.
